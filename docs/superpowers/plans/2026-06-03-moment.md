# Moment Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the new REST-style moment backend while preserving old say behavior.

**Architecture:** Add a focused moment module across DTO, repository, service, handler, router, and Swagger. Reuse existing `moment`, `media`, `user_like`, `moment_comment`, `message`, and `user_message` tables.

**Tech Stack:** Go 1.25, Gin, GORM/MySQL, go-sqlmock, testify, swaggo.
