# Tag API Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build tag CRUD, tag article relation maintenance, and paginated article listing under a tag.

**Architecture:** Mirror the existing category module for tag DTOs, service, repository, handler, and router registration. Reuse article service for `GET /tags/:id/articles` so article pagination and response mapping stay consistent with `/articles?tag_id=`.

**Tech Stack:** Go 1.25, Gin, GORM/MySQL, go-sqlmock, gomock, httptest, swaggo.

---

## File Structure

- Modify `internal/dto/category.go`: add tag request and response DTOs near category DTOs.
- Modify `internal/repository/tag.go`: define tag repository interface, aggregate structs, constructor, queries, and mutations.
- Modify `internal/service/tag.go`: define tag service interface, validation, error mapping, DTO conversion, and article listing delegation.
- Modify `internal/handler/tag.go`: implement public and admin HTTP endpoints with Swagger annotations.
- Modify `internal/router/router.go`: wire tag repository/service/handler and register public/admin routes.
- Modify `internal/repository/mock/mock_category_repository.go`: keep existing generated mock intact.
- Create `internal/repository/mock/mock_tag_repository.go`: gomock-compatible tag repository mock.
- Create `internal/service/tag_test.go`: service tests using gomock.
- Create `internal/repository/tag_test.go`: sqlmock tests.
- Create `internal/handler/tag_test.go`: httptest handler tests.
- Modify generated Swagger files via `make swag`.
